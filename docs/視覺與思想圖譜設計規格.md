版本：MVP 第一版  
角色：聊天室 D，視覺與思想圖譜設計師

## 一、核心結論

本網站的頁面風格應以「學術資料庫的秩序」與「思想地圖的可探索性」為主軸，而非以品牌展示或內容敘事作為第一層邏輯。

MVP 視覺系統採取三項原則：

1. 以黑白灰、細線、網格、時間刻度建立穩定的知識介面。
2. 以低飽和藍綠色作為唯一強調色，用於選取、焦點、活躍關係與互動回饋。
3. 以 Philosopher、School、Concept、Work、InfluenceRelation 五類資料作為視覺語言的基礎單位。

整體介面應像一座可查詢的哲學圖書館，也像一張可縮放的思想索引圖。此處「地圖」是功能性隱喻，表示使用者能看見位置、關係、方向與鄰近性，不表示必須製作地理地圖。

## 三、色彩系統

### 3.1 基礎色

| Token | 色值 | 用途 |
|---|---:|---|
| `--color-ink` | `#111111` | 主文字、主要節點、重要邊框 |
| `--color-text` | `#2A2A2A` | 內文 |
| `--color-muted` | `#6E6E6E` | 次要資訊、年代、輔助標籤 |
| `--color-subtle` | `#A8A8A8` | 刻度、次級線條、停用文字 |
| `--color-line` | `#D8D8D8` | 一般邊框與分隔線 |
| `--color-grid` | `#ECECEC` | 背景網格 |
| `--color-surface` | `#FAFAFA` | 頁面背景 |
| `--color-panel` | `#FFFFFF` | 工具列、列表、資訊面板 |

### 3.2 強調色

| Token | 色值 | 用途 |
|---|---:|---|
| `--color-accent` | `#3E6F73` | 主要互動、選取節點、活躍篩選 |
| `--color-accent-soft` | `#DDE9E8` | 選取背景、hover 面 |
| `--color-accent-line` | `#8DB2B0` | 關係圖高亮線、時間軸焦點 |

強調色只允許用於互動狀態、資料選取與關係提示。不得用於大面積背景，以免破壞資料庫介面的冷靜性。

### 3.3 關係類型色彩

| Relation type | 線條處理 |
|---|---|
| `direct` | 實線，`--color-ink` |
| `critique` | 實線加短節點標記，`--color-accent` |
| `reinterpretation` | 長虛線，`--color-accent-line` |
| `textual` | 點線，`--color-muted` |
| `indirect` | 細虛線，`--color-subtle` |
| `disputed` | 雙虛線，附 `?` 標記 |

## 五、排版規則

### 5.1 桌機版基本格線

桌機版使用 12 欄格線：

1. 最大內容寬度：1440px。
2. 主要邊距：32px 至 48px。
3. 欄距：24px。
4. 右側資訊面板寬度：320px 至 380px。
5. 左側篩選欄寬度：260px 至 300px。

頁面應避免把每個區塊都做成浮動卡片。主要結構以全寬資料帶、分隔線、列表、面板與圖譜區構成。

### 5.2 間距系統

| Token | 數值 | 用途 |
|---|---:|---|
| `--space-1` | 4px | 細部間距 |
| `--space-2` | 8px | 標籤與小元件 |
| `--space-3` | 12px | 表單內距 |
| `--space-4` | 16px | 元件間距 |
| `--space-6` | 24px | 區塊內距 |
| `--space-8` | 32px | 區塊間距 |
| `--space-12` | 48px | 主區段間距 |

### 5.3 資訊層級

每一頁必須遵守「先定位，後展開，再連接」：

1. 定位：名稱、時代、文明、流派、概念。
2. 展開：summary、keyQuestions、keyIdeas、works。
3. 連接：influencedBy、influenced、relatedConcepts、relatedSchools。

## 六、線條與邊框規則

### 6.1 線條

1. 一般分隔線：1px，`--color-line`。
2. 背景網格線：1px，`--color-grid`，透明度 60%。
3. 關係線：1px 至 1.5px。
4. 活躍關係線：2px，`--color-accent`。
5. 時間軸主線：1px，`--color-ink`。
6. 時間軸刻度線：1px，`--color-subtle`。

### 6.2 邊框與圓角

1. 一般元件圓角：4px。
2. 節點圓角：999px 或 4px，依節點類型決定。
3. 卡片圓角：6px，最高不得超過 8px。
4. 不使用厚陰影。若必須區分浮層，使用 1px 邊框與輕微背景差。

### 6.3 背景網格

思想地圖與時間軸可使用 24px 網格：

```css
background-image:
  linear-gradient(var(--color-grid) 1px, transparent 1px),
  linear-gradient(90deg, var(--color-grid) 1px, transparent 1px);
background-size: 24px 24px;
```

網格是定位工具，不是裝飾主體；文字密集區不使用高對比網格背景。

## 七、元件視覺規範

### 7.1 按鈕

| 狀態 | 規格 |
|---|---|
| Default | 白底、1px 灰線、文字 `--color-text` |
| Hover | 背景 `--color-accent-soft`、邊框 `--color-accent-line` |
| Active | 背景 `--color-accent`、文字白色 |
| Disabled | 文字 `--color-subtle`、背景 `--color-surface` |

按鈕高度：

1. 小型：32px。
2. 一般：40px。
3. 工具列 icon button：36px 正方形。

### 7.2 搜尋框

搜尋框是首頁與列表頁的主要入口。

1. 高度：48px。
2. 邊框：1px `--color-ink`。
3. 左側放搜尋圖示。
4. placeholder 使用 `--color-muted`。
5. focus 時邊框改為 `--color-accent`，背景維持白色。
6. 支援搜尋中文名、原文名、流派、概念、著作。

### 7.3 篩選器

篩選器採「分類面板 + chip」形式：

1. 分類：Era、Region、Civilization、School、Concept、Field。
2. 同一分類內多選為 OR。
3. 不同分類之間為 AND。
4. active chip 使用 `--color-accent` 邊框與淡色背景。
5. 已選篩選器集中顯示於結果列表上方。

### 7.4 哲學家列表卡片

卡片不是厚重容器，而是可掃描的資料列。

內容順序：

1. 中文名與原文名。
2. 生卒年與時代。
3. 文明、地區、主要流派。
4. oneLine 或 summary 短句。
5. 主要概念 chip。
6. 收藏、比較、查看關係按鈕。

視覺規則：

1. 白底。
2. 底部 1px 分隔線。
3. hover 時左側出現 2px accent 導引線。
4. 不使用大陰影。

### 7.5 標籤

標籤分為三類：

1. Era：等寬字體，細框。
2. School：矩形 chip。
3. Concept：膠囊 chip。

Concept chip 可用於反向探索。點擊後進入概念索引或套用概念篩選。

### 7.6 節點

| 節點類型 | 形狀 | 尺寸 | 文字 |
|---|---|---:|---|
| Philosopher | 圓形或圓角矩形 | 48 至 72px | 中文名 |
| School | 矩形 | 96 x 32px | 流派名 |
| Concept | 膠囊 | 自適應 | 概念名 |
| Work | 小矩形 | 88 x 28px | 書名 |

核心哲學家節點使用較高對比；鄰近節點使用淡線與較低對比。不得一次顯示過多文字造成重疊。

## 八、首頁思想地圖設計

### 8.1 首頁第一屏目標

首頁第一屏要立即傳達：這是一個可搜尋、可篩選、可比較、可探索關係的世界哲學思想資料庫。

第一屏結構：

1. 左側：網站名稱、定位句、搜尋框、主要篩選入口。
2. 右側：思想節點地圖。
3. 底部：Era / Civilization 快速索引條。

### 8.2 思想節點地圖

MVP 首頁不做完整全域網絡。建議顯示「代表性節點群」：

1. 古希臘羅馬：蘇格拉底、柏拉圖、亞里斯多德。
2. 中華文明：孔子、孟子、老子、莊子。
3. 印度與佛教：釋迦牟尼、龍樹、商羯羅。
4. 近代歐洲：笛卡兒、休謨、康德、黑格爾。
5. 二十世紀：維根斯坦、鄂蘭、傅柯。

節點以文明群聚，並以時間方向由左至右排列。關係線只顯示高確定性關係，避免首頁變成不可讀的大型網。

### 8.3 首頁互動

1. hover 哲學家節點：高亮該節點、顯示 oneLine、時代、文明、主要概念。
2. click 哲學家節點：開啟右側簡介面板，可進入詳情頁。
3. hover 關係線：顯示 relationType、summary、evidenceLevel。
4. click civilization 群組：套用文明篩選並跳到列表。
5. 搜尋輸入後：地圖節點同步淡化不匹配項。

## 九、世界哲學時間軸設計

### 9.1 桌機版

桌機版採橫向時間軸：

1. X 軸：年份，西元前使用負數資料轉換顯示。
2. Y 軸：Civilization 或 Region 分層。
3. 節點：哲學家。
4. 區段背景：Era。
5. 不確定年代：節點外框使用虛線，年份前顯示「約」。

時間軸上方放 Era 快速切換，下方放哲學家資料列。點擊節點時，下方資料列同步定位。

### 9.2 手機版

手機版不使用橫向大時間軸。替代方案：

1. 垂直年代列表。
2. 以 Era 作為折疊區段。
3. 每個哲學家以 compact row 呈現。
4. 支援「依文明分組」與「依時間排序」切換。
5. 關係線不直接繪製，改用「受誰影響 / 影響誰」列表。

## 十、哲學家影響關係圖設計

### 10.1 MVP 範圍

第一版只做「單一哲學家的鄰近關係圖」，不做全站自由縮放網絡。

中心節點為當前哲學家：

1. 左側：influencedBy。
2. 中央：current philosopher。
3. 右側：influenced。
4. 下方：shared concepts。
5. 上方：related schools / works。

### 10.2 關係線

線條方向由 source 指向 target。箭頭應清楚，但不可過大。

關係線標籤顯示：

1. relationType。
2. strength。
3. evidenceLevel。

hover 線條時顯示 summary。若 evidenceLevel 為 `scholarly-debate` 或 `speculative`，線條使用虛線並顯示註記。

### 10.3 Click 行為

1. 點擊哲學家節點：導向該哲學家詳情頁。
2. 點擊概念節點：套用概念篩選或進入概念頁。
3. 點擊著作節點：開啟著作摘要面板。
4. 點擊關係線：開啟 InfluenceRelation 詳細說明。

## 十一、流派與概念關聯圖設計

### 11.1 流派頁

流派頁應呈現「一組問題意識與方法」而非普通標籤頁。

頁面結構：

1. 流派名稱、originalName、era、civilization。
2. definition 與 coreIdeas。
3. 關聯哲學家列表。
4. 關聯概念列表。
5. 關聯著作。
6. relatedSchoolIds。

### 11.2 概念頁

概念頁應呈現「概念在不同哲學家與流派中的分布」。

頁面結構：

1. 概念名稱與 field。
2. definition。
3. 相關哲學家。
4. 相關流派。
5. 相關著作。
6. relatedConceptIds 與 oppositeConceptIds。

### 11.3 關聯圖

關聯圖採三層結構：

1. 左層：School。
2. 中層：Concept。
3. 右層：Philosopher / Work。

使用者選取一個 Concept 時，只高亮與它直接相關的 School、Philosopher、Work，其餘節點淡化。

## 十二、互動狀態定義

### 12.1 Hover

Hover 不改變資料狀態，只提供局部解釋。

1. 節點：加粗邊框、顯示 tooltip。
2. 線條：加粗、顯示 relation summary。
3. chip：背景淡化、游標 pointer。
4. 卡片：左側 accent 導引線。

### 12.2 Click

Click 必須產生明確狀態：

1. 節點被選取。
2. 右側或底部面板開啟。
3. URL 或篩選狀態可追蹤。
4. 可用 Esc 或關閉按鈕回到未選取狀態。

### 12.3 Filter

篩選後需要同步更新：

1. 列表結果數量。
2. 首頁或圖譜節點可見性。
3. 已選條件列。
4. 空狀態文案。

空狀態文案應提供可操作方向，例如「移除部分流派或概念篩選」。

### 12.4 View switch

列表、時間軸、圖譜三種視圖切換應維持同一組篩選條件。

視圖切換器：

1. List。
2. Timeline。
3. Map。

手機版可簡化為：

1. List。
2. Timeline。
3. Relations。

## 十三、桌機版視覺規範

### 13.1 首頁

1. 首屏高度約 82vh，底部露出下一區段。
2. 不使用大型 hero 圖片。
3. 以搜尋框、節點圖、快速篩選作為首頁核心。
4. 第一屏下方接「近期可探索的文明 / 時代 / 概念」索引。

### 13.2 列表頁

1. 左側固定篩選欄。
2. 中央資料列表。
3. 右側可選比較欄或收藏欄。
4. 排序控制放在結果標題列。

### 13.3 詳情頁

詳情頁採三欄：

1. 左欄：頁內目錄、基本資料、收藏與比較。
2. 中欄：summary、biography、keyQuestions、keyIdeas、works。
3. 右欄：鄰近關係圖、related concepts、influence links。

### 13.4 比較頁

比較頁採欄式矩陣：

1. 欄：哲學家。
2. 列：可比較欄位。
3. 差異重點用細線與淡色背景標示。
4. 缺資料使用中性 fallback，不留空白。

## 十四、手機版視覺規範

### 14.1 原則

手機版不是把桌機圖譜縮小，而是改以「列表化、分段化、抽屜式篩選」呈現同一套資料關係。

### 14.2 導航

1. 底部固定導覽：搜尋、列表、時間、收藏。
2. 篩選器改為 bottom sheet。
3. 搜尋框置頂，可收合。

### 14.3 圖譜替代方案

1. 首頁思想地圖改為文明分組橫向節點列。
2. 影響關係圖改為上下游列表。
3. 概念關聯圖改為 expandable relation sections。
4. 時間軸改為垂直清單。

### 14.4 可用性

1. 觸控目標不小於 44px。
2. chip 可換行，不擠壓文字。
3. 長原文名以兩行內截斷，保留 tooltip 或詳情展開。
4. 不在小螢幕上顯示超過兩層關係線。

前端不得從中文 summary 推導關係。所有關係應讀取 `InfluenceRelation` 與 id 欄位。

## 十六、QA 檢查重點

1. 黑白灰與 accent 色是否維持克制比例。
2. 手機版是否沒有把桌機大型圖譜硬縮小。
3. 篩選條件在 List、Timeline、Map 間是否維持一致。
4. hover 與 click 是否有明確差異。
5. 西元前年份是否排序正確。
6. 年代不確定資料是否有「約」或虛線標記。
7. 影響關係方向是否遵守 source -> target。
8. 長中文名與長英文名是否不溢出。
9. 空狀態與缺資料 fallback 是否穩定。
10. 收藏與比較是否只儲存 id。

## 十七、下一個角色需要接收的重點摘要

交給前端工程師：

1. 頁面風格不是 landing page，而是資料庫與思想地圖混合介面。
2. MVP 圖譜只做局部鄰近關係，不做全域大型網絡。
3. 視覺基礎是黑白灰、細線、網格、時間刻度與唯一低飽和 accent。
4. 桌機可使用三欄與圖譜面板；手機需改為列表、bottom sheet 與折疊關係。
5. 圖譜必須讀取 `InfluenceRelation`，不可從文字摘要推論。

交給 QA Tester：

1. 優先檢查資料關聯方向、時間排序、篩選狀態同步與手機版替代呈現。
2. 視覺品質重點不是華麗程度，而是可讀、可比較、可探索、可擴充。
3. 所有互動狀態都必須能被清楚復原或關閉。
